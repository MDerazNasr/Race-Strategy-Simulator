D28 is the direct fix for d27's LR bug. D27 confirmed that pit_timing_reward=True
is SAFE with frozen driving, but the pit row barely moved (+1.1%) because SB3's
reset_num_timesteps=False caused the LR to start at 3.5e-5 instead of 1e-4.

D28 applies the same setup as d27, but with reset_num_timesteps=True so the cosine
LR schedule restarts from progress=1.0 (LR = 1e-4 at step 0 of d28).

SETUP:
  Starting point: ppo_pit_v4_d27.zip (d27 — most-shaped pit row, bias=-0.839)
  Environment: make_env_pit_d23 (pit_timing_reward=True)
    - tyre_life < 0.30 when pitting: +100 bonus (net cost = -100)
    - tyre_life > 0.50 when pitting: -100 extra (net cost = -300)
    - Neutral zone 0.30–0.50: no change (net cost = -200)
  Three-layer freeze: identical to d26/d27
    Layer 1: mlp_extractor.policy_net (requires_grad=False, 18,176 params)
    Layer 2: action_net.weight rows 0,1 + bias[0,1] (gradient hooks)
    Layer 3: log_std[0,1] (gradient hook)
  Only 130 policy params trainable: action_net.weight[2,:] + bias[2] + log_std[2]
  STAGES_PIT_V5: safety net 0.25→0.15→0.08
  LR: cosine 1e-4→1e-6 with reset_num_timesteps=True (THE KEY FIX)
  Total steps: 2M

THE KEY FIX vs D27:
  D27: reset_num_timesteps=False → SB3 computes progress = 1 - 3M/5M = 0.40
       → cosine(1e-4, 1e-6, 0.40) ≈ 3.5e-5 at step 0 of d27 (WRONG)
  D28: reset_num_timesteps=True  → SB3 treats d28 as fresh → progress = 1.0
       → LR starts at exactly 1e-4 (confirmed in training log: 9.96e-05)

ROOT CAUSE ANALYSIS: WHY FIXED-START IS STILL 1883
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  Pattern: d26=1883.41, d27=1883.41, d28=1883.41 on fixed-start.
  The pit row IS moving (bias: -0.817 → -0.839 → -0.929 across d26/d27/d28).
  The pit_timing_reward gradient is doing something — but it's not showing up as
  a better fixed-start result.

  HYPOTHESIS A — Episode ends before delayed pit fires:
    The fixed-start episode terminates at step 1354 (crash or worn tyres).
    The agent's pit firing threshold is shifting toward tyre_life=0.30 from 0.35.
    If the natural pit moment shifts from step X to step X+50, but the episode
    ends at step X+30, the delayed pit never fires → same reward trajectory.

  HYPOTHESIS B — Pit timing already near-optimal for the fixed-start trajectory:
    Given the fixed-start dynamics, the current pit timing (tyre_life≈0.35) may
    already be close to the best achievable within the episode structure.
    The +100 timing bonus only fires when the agent pits at tyre_life < 0.30,
    but if the episode ends before tyre_life reaches 0.30, the bonus is unreachable.

  HYPOTHESIS C — More training needed:
    The bias trend (-0.817 → -0.839 → -0.929) is consistent and moving in the
    right direction. Another 2M steps with the same setup might push bias to
    -1.1 or -1.2, shifting the pit boundary enough to earn the timing bonus.

  EVIDENCE SUPPORTING A/B:
    Fixed-start ep always ends at step 1354 (consistent termination point).
    Pit fires at approximately the same step each time → no variability.
    The fixed-start run may be "stuck" in a phase-space trajectory where the
    timing bonus is geometrically unreachable without more fundamental changes.
