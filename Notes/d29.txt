==============================================================================
ROOT CAUSE ANALYSIS — WHY D29 FAILED (SAME CEILING)
==============================================================================

D29 confirmed a fundamental flaw in the forced-pit curriculum for training pit_signal:

THE GRADIENT PROBLEM:
  The forced_pit_threshold mechanism fires the pit REGARDLESS of the agent's pit_signal.
  PPO computes advantage for the agent's CHOSEN action (pit_signal ≈ -1.0 due to bias).

  Sequence:
  1. tyre_life < 0.50 → forced pit fires (regardless of agent's pit_signal)
  2. Agent had output pit_signal ≈ -0.5 (bias + features)
  3. After forced pit: agent survives to step 2000, gets +800 more reward
  4. Advantage for (state, action=[throttle, steer, pit_signal=-0.5]) is POSITIVE
  5. PPO gradient: "low pit_signal at worn-tyre state → positive advantage → REINFORCE low pit_signal"
  6. Bias moves MORE NEGATIVE: +0.006 → -1.217

The value function learns correctly (Q(worn_tyre, pit) >> Q(worn_tyre, no_pit)).
The POLICY gradient goes in the WRONG direction (reinforces low pit_signal).

WHY D21 WORKED (ONE TIME):
  D21 loaded BC-initialized weights. The BC features happened to encode tyre_life
  in a way that made pit_signal slightly POSITIVE at worn-tyre states. When the
  forced pit fired and the agent's pit_signal was positive → gradient reinforced
  positive pit_signal → voluntary pitting learned. This was essentially LUCKY:
  the BC feature encoding aligned with the forced pit timing.

D26/D27/D28/D29 ALL STARTED FROM D21 (pit_bias=+0.006) BUT FAILED because:
  - When forced pit fires, the agent's pit_signal was net negative (even at +0.006 bias,
    the features pushed it negative at the specific worn-tyre states)
  - Gradient reinforced negative pit_signal consistently
  - Bias spiraled: d21=+0.006 → d26=-0.817 → d27=-0.839 → d28=-0.929 → d29=-1.217

EVIDENCE: Pit weight doubled (+183%) but in the wrong direction (negative bias).
The agent became MORE "opinionated" about not pitting, just more strongly negative.

==============================================================================
WHAT'S NEXT (d30): FIX THE GRADIENT DIRECTION
==============================================================================

The forced-pit curriculum must be redesigned to train pit_signal CORRECTLY.
The core requirement: when forced pit fires, the stored (state, action) must have
pit_signal > 0. Then advantage flows back to high pit_signal → correct gradient.

APPROACH OPTIONS:

Option A — BC Pre-training of Pit Row:
  1. Load d21 features (frozen)
  2. Collect states from d21 rollouts at various tyre_life levels
  3. Run supervised learning ONLY on action_net.weight[2,:] + bias[2]:
       - Target: pit_signal = +2.0 when tyre_life < 0.55
       - Target: pit_signal = -2.0 when tyre_life > 0.70
  4. Save BC-initialized pit row
  5. Run PPO fine-tuning (same freeze setup)
  Result: Pit row enters RL pre-initialized to output high pit_signal at worn states.
  When forced pit fires, stored pit_signal > 0 → gradient reinforces correctly.

Option B — Intrinsic Reward for Pit Action:
  In F1Env / reward fn: when pit fires AND agent's pit_signal > 0 (voluntary):
    reward += R_voluntary (e.g., +300)
  When tyre_life < threshold AND pit_signal < 0 (missed opportunity):
    reward -= per_step_penalty (e.g., -0.5/step)
  Remove forced pit entirely — rely on reward shaping to drive exploration.
  This directly rewards the CHOICE to pit, not just the result of pitting.

Option C — Rollout Action Override:
  Custom PPO subclass that post-processes the rollout buffer: when tyre_life < threshold,
  replace stored action's pit_signal with +1.0. The advantage then flows back to
  high pit_signal. Cleanest mechanistically, hardest to implement in SB3.

RECOMMENDATION: Option B (intrinsic reward) is simplest and most principled.
  - No SB3 internals to override
  - Directly trains: "choosing to pit at worn tyre → high reward"
  - Works with stochastic exploration (high pit_std means agent sometimes outputs
    pit_signal > 0, gets rewarded, gradient reinforces further)
  - Key: R_voluntary must be large enough to overcome -200 pit penalty

For d30, try: R_voluntary = +300 when voluntary pit fires at tyre_life < 0.50.
Load d21 (near-zero bias). No forced pit. Monitor if agent discovers voluntary pitting
within first 200k steps (when pit_std is still high enough for exploration).
